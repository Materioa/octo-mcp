# Diagram Tool Usage Examples

## Quick Reference

### Generate → Render Workflow

```
User Request
    ↓
GenerateDiagramFromRequest (get template)
    ↓
[LLM generates diagram spec]
    ↓
Choose render path:
- DiagramGenerator for Mermaid/DOT JS rendering
- Web-interface sandbox for Python-heavy diagrams
    ↓
Base64 PNG / File / SVG output
```

### Output Handling (Platform-Specific)

- ChatGPT/Claude: Mermaid code blocks render natively.
- ChatGPT/Claude: SVG renders natively via HTML artifact.
- ChatGPT/Claude: HTML output must be returned as an artifact to trigger native rendering.
- ChatGPT/Claude: Raw SVG text should be wrapped in a minimal HTML artifact to render. Do not return JSON-wrapped SVG text in chat.
- ChatGPT/Claude: Do not trim SVG output before embedding.
- ChatGPT/Claude: SVG/PNG outputs should be provided as download links (do not try to render inline).
- ChatGPT/Claude: When tool output is DOT/PNG, render via sandbox Python and provide a download link. Only render natively if the response is Mermaid or SVG.
- ChatGPT/Claude: Do not skip diagram rendering and do not answer with text or ASCII drawings.
- Save images to /mnt/data or ./output and return a download link or artifact pointing to that file.
- ChatGPT: prefer python_user_visible so the image renders inline in chat.
- Perplexity: Always render diagrams in the sandbox with Python and return a download link only. Do not inline or describe.
- Never convert DOT to Mermaid or present DOT output as Mermaid.
- If the user asked to draw or render a diagram, do not answer with text-only tables or descriptions. Always return a rendered diagram image.
- If a tool returns SVG/XML, you must pass it into the sandbox exactly as received, without truncating or trimming anything, render to PNG, and return the image output. Do not answer until the image is generated; no text or ASCII fallback.
- For all platforms: if native rendering is unavailable or fails, use the platform sandbox to render the diagram into an image.
- For any diagram type other than Mermaid or DOT (plots, visualizations, circuits, Gantt charts, logic gates, timelines), always use the sandbox with Python and return the image output. No ASCII drawings.
- Image output has priority 1.
- Text, tables, and explanations have priority 3.
- Never let text outrank image output when a diagram was requested.

## Example 1: FSM Diagram (DOT Format)

### Request
"Create a DFA that accepts strings over {0,1} ending with '01'"

### Step 1: GenerateDiagramFromRequest
```
Input:  request="DFA that accepts strings ending with 01"
        format="auto"
Output: Recommends DOT format with template
```

### Step 2: Generate DOT Specification
```dot
digraph DFA {
  rankdir=LR;
  node [shape=circle, style=filled, fillcolor="#e8f4fd", color="#2196F3", fontname="Arial"];
  edge [color="#555555", fontname="Arial", fontsize="10"];
  
  q0 [label="q₀\n(start)"];
  q1 [label="q₁\n(seen 0)"];
  q2 [label="q₂\n(accept)", shape=doublecircle];
  
  // Self-loops and transitions
  q0 -> q0 [label="1"];
  q0 -> q1 [label="0"];
  q1 -> q1 [label="0"];
  q1 -> q2 [label="1"];
  q2 -> q0 [label="1"];
  q2 -> q1 [label="0"];
}
```

### Step 3: Render with DiagramGenerator
```
Input:
  format="dot"
  spec="<above dot code>"
  render="png"

Output:
  PNG image at /tmp/diagram_1234567890.png
  OR base64 data URI for embedding
```

### Result
Professional, accurate FSM diagram that renders perfectly across all platforms.

### Perplexity Sandbox (Python Graphviz) Example
```python
import sys
import subprocess

try:
  import graphviz
except ImportError:
  subprocess.check_call([sys.executable, "-m", "pip", "install", "graphviz"])
  import graphviz

dot = graphviz.Digraph(format="png")
dot.attr(rankdir="LR")
dot.attr("node", shape="circle")

dot.node("q0", "q0")
dot.node("q1", "q1")
dot.node("q2", "q2", shape="doublecircle")

dot.edge("q0", "q0", label="1")
dot.edge("q0", "q1", label="0")
dot.edge("q1", "q1", label="0")
dot.edge("q1", "q2", label="1")
dot.edge("q2", "q1", label="0")
dot.edge("q2", "q0", label="1")

dot.render("/tmp/diagram", cleanup=True)
# Output: /tmp/diagram.png
```

## Example 2: Circuit Diagram

### Request
"Draw an RC low-pass filter circuit with R=1kΩ and C=1µF"

### Step 1: GenerateDiagramFromRequest
```
Input:  request="RC low-pass filter circuit"
        format="auto"
Output: Recommends circuit format with Python template
```

### Step 2: Generate Python Circuit Code
```python
import schemdraw
import schemdraw.elements as elm

with schemdraw.Drawing() as d:
    # Title
    d += (R := elm.Resistor().label('R=1kΩ', loc='top'))
    
    # Connection node
    d += elm.Line().right(0.5)
    d.push()
    
    # Capacitor to ground
    d += (C := elm.Capacitor().down().label('C=1µF', loc='right'))
    d += elm.Ground()
    
    # Output tap
    d.pop()
    d += elm.Line().right(0.5)
    d += elm.Gap().label(['Vout', '−'], loc='right')
    
    # Save output
    d.save('/tmp/diagram.png')
```

### Step 3: Run in the Web-Interface Sandbox
Execute the Python code in the ChatGPT, Claude, or Perplexity sandbox to produce the diagram image. Do not send Python code to a server-side MCP sandbox.

## Example 3: Data Visualization (Matplotlib)

### Request
"Create a line plot showing sin(x) and cos(x) from 0 to 2π"

### Step 1: GenerateDiagramFromRequest
```
Input:  request="Line plot of sin and cos"
        format="auto"
Output: Recommends plot format with template
```

### Step 2: Generate Python Plot Code
```python
import matplotlib.pyplot as plt
import numpy as np

# Generate data
x = np.linspace(0, 2*np.pi, 200)
y_sin = np.sin(x)
y_cos = np.cos(x)

# Create figure
fig, ax = plt.subplots(figsize=(12, 6))

# Plot lines
ax.plot(x, y_sin, linewidth=2.5, label='sin(x)', color='#FF6B6B')
ax.plot(x, y_cos, linewidth=2.5, label='cos(x)', color='#4ECDC4')

# Formatting
ax.grid(True, alpha=0.3, linestyle='--')
ax.set_xlabel('x (radians)', fontsize=12)
ax.set_ylabel('y', fontsize=12)
ax.set_title('Trigonometric Functions', fontsize=14, fontweight='bold')
ax.legend(fontsize=11, loc='upper right')
ax.set_xlim(0, 2*np.pi)
ax.set_ylim(-1.5, 1.5)

# Save
plt.savefig('/tmp/diagram.png', dpi=150, bbox_inches='tight', transparent=True)
plt.close()
```

### Result
High-quality visualization perfect for presentations and documentation.

## Example 4: Gantt Chart (Project Timeline)

### Request
"Gantt chart for a 10-week software development project"

### Step 1: Generate Python Gantt Code
```python
import matplotlib.pyplot as plt
from datetime import datetime, timedelta

# Define tasks
tasks = [
    {'name': 'Requirements & Planning', 'start': 0, 'duration': 2, 'color': '#FF6B6B'},
    {'name': 'Architecture & Design', 'start': 2, 'duration': 3, 'color': '#4ECDC4'},
    {'name': 'Backend Development', 'start': 3, 'duration': 5, 'color': '#45B7D1'},
    {'name': 'Frontend Development', 'start': 4, 'duration': 4, 'color': '#FFA07A'},
    {'name': 'Testing & QA', 'start': 8, 'duration': 2, 'color': '#95E1D3'},
    {'name': 'Deployment & Launch', 'start': 10, 'duration': 1, 'color': '#F38181'},
]

fig, ax = plt.subplots(figsize=(14, 7))

# Draw bars
for i, task in enumerate(tasks):
    ax.barh(i, task['duration'], left=task['start'], height=0.6,
            color=task['color'], edgecolor='black', linewidth=1.5,
            label=task['name'])

# Formatting
ax.set_yticks(range(len(tasks)))
ax.set_yticklabels([t['name'] for t in tasks], fontsize=11)
ax.set_xlabel('Weeks', fontsize=12, fontweight='bold')
ax.set_title('Software Development Project Timeline', fontsize=14, fontweight='bold')
ax.grid(True, alpha=0.3, axis='x', linestyle='--')
ax.set_xlim(0, 11)

plt.tight_layout()
plt.savefig('/tmp/diagram.png', dpi=150, bbox_inches='tight', transparent=True)
plt.close()
```

### Result
Clear project timeline with task dependencies and duration visualization.

## Example 5: Logic Gates Circuit

### Request
"Draw a full adder circuit using AND, OR, and XOR gates"

### Step 1: Generate Python Logic Gate Code
```python
import matplotlib.pyplot as plt
import matplotlib.patches as patches
from matplotlib.patches import FancyBboxPatch, Circle, Polygon, FancyArrowPatch

fig, ax = plt.subplots(figsize=(14, 8))
ax.set_xlim(0, 14)
ax.set_ylim(0, 8)
ax.axis('off')

def draw_gate(ax, x, y, gate_type, label='', size=0.8):
    """Draw a logic gate"""
    if gate_type == 'XOR':
        # XOR gate (D-shape with double curve)
        gate = FancyBboxPatch((x-0.5, y-0.5), 1, 1, boxstyle="round,pad=0.05",
                             edgecolor='black', facecolor='lightblue', linewidth=2)
        ax.add_patch(gate)
        ax.text(x, y, 'XOR', ha='center', va='center', fontsize=9, fontweight='bold')
    elif gate_type == 'AND':
        # AND gate
        gate = FancyBboxPatch((x-0.4, y-0.4), 0.8, 0.8, boxstyle="round,pad=0.05",
                             edgecolor='black', facecolor='lightgreen', linewidth=2)
        ax.add_patch(gate)
        ax.text(x, y, 'AND', ha='center', va='center', fontsize=8, fontweight='bold')
    elif gate_type == 'OR':
        # OR gate
        gate = FancyBboxPatch((x-0.4, y-0.4), 0.8, 0.8, boxstyle="round,pad=0.05",
                             edgecolor='black', facecolor='lightyellow', linewidth=2)
        ax.add_patch(gate)
        ax.text(x, y, 'OR', ha='center', va='center', fontsize=8, fontweight='bold')

def draw_wire(ax, x1, y1, x2, y2):
    """Draw a connection wire"""
    ax.plot([x1, x2], [y1, y2], 'k-', linewidth=2)

def draw_input_label(ax, x, y, label):
    """Draw input label"""
    ax.text(x-0.3, y, label, ha='right', va='center', fontsize=10, fontweight='bold')

def draw_output_label(ax, x, y, label):
    """Draw output label"""
    ax.text(x+0.3, y, label, ha='left', va='center', fontsize=10, fontweight='bold')

# Inputs
draw_input_label(ax, 1, 6.5, 'A')
draw_input_label(ax, 1, 5.5, 'B')
draw_input_label(ax, 1, 4.5, 'Cin')

# First XOR gate (A XOR B)
draw_gate(ax, 3, 6, 'XOR')
draw_wire(ax, 1.5, 6.5, 2.5, 6.2)
draw_wire(ax, 1.5, 5.5, 2.5, 5.8)
draw_wire(ax, 3.5, 6, 4.5, 6)

# Second XOR gate (result XOR Cin)
draw_gate(ax, 6, 6, 'XOR')
draw_wire(ax, 4.5, 6, 5.5, 6.2)
draw_wire(ax, 1.5, 4.5, 5.5, 5.8)
draw_output_label(ax, 6.5, 6, 'Sum')

# AND gates for carry
draw_gate(ax, 3, 4, 'AND')
draw_wire(ax, 1.5, 6.5, 2.5, 4.2)
draw_wire(ax, 1.5, 5.5, 2.5, 3.8)
draw_wire(ax, 3.5, 4, 4.5, 4)

draw_gate(ax, 3, 2.5, 'AND')
draw_wire(ax, 4.5, 6, 2.5, 2.7)
draw_wire(ax, 1.5, 4.5, 2.5, 2.3)
draw_wire(ax, 3.5, 2.5, 4.5, 2.5)

# Final OR gate for Cout
draw_gate(ax, 6, 3.2, 'OR')
draw_wire(ax, 4.5, 4, 5.5, 3.4)
draw_wire(ax, 4.5, 2.5, 5.5, 3)
draw_output_label(ax, 6.5, 3.2, 'Cout')

# Title
ax.text(7, 7.5, 'Full Adder Circuit', ha='center', fontsize=14, fontweight='bold')

plt.savefig('/tmp/diagram.png', dpi=150, bbox_inches='tight', transparent=True)
plt.close()
```

### Result
Complete full adder circuit with clearly labeled inputs, outputs, and gate operations.

## Tips for Each Format

### DOT (FSMs, Graphs)
- Always set `rankdir` (LR, TB, RL, BT)
- Define all nodes before edges
- Use `shape=doublecircle` for accepting states
- Clear, concise labels on edges

### Mermaid (Flowcharts, State Machines)
- Good for general diagrams but less accurate than DOT for FSMs
- Use when visual clarity is priority over strict accuracy
- Supports multiple diagram types

### Circuit (Electronics)
- Use schemdraw for professional schematics
- Name components clearly (R1, C1, etc.)
- Use proper units (Ω, µF, mA, etc.)

### Plot (Visualizations)
- Set DPI to 150+ for quality
- Use `transparent=True` for clean backgrounds
- Add grid and labels for clarity
- Use consistent color schemes

### Gantt (Timelines)
- Start dates and duration must be numeric
- Use descriptive task names
- Consider color-coding by category
- Include grid lines for readability

### Logic (Boolean Circuits)
- Be explicit about gate types (AND, OR, XOR, NAND, NOR, NOT)
- Label inputs/outputs clearly
- Show truth table alongside diagram if complex
- Keep gate arrangements clean and logical

## Error Handling Examples

### Error: "DOT syntax error"
```
Cause: Missing semicolon or invalid node name
Fix: Add semicolon; use quoted names with spaces
```

### Error: "Python diagram execution failed"
```
Cause: Missing savefig(), wrong output path, or sandbox package mismatch
Fix: Ensure output file is /tmp/diagram.png and adapt imports to the target web-interface sandbox
```

### Error: "SVG to PNG conversion failed"
```
Cause: Invalid SVG or unsupported features
Fix: Use DiagramValidator first; try SVG output instead
```

## Advanced Usage

### Combining Multiple Diagrams
```
Request: "Show both DFA and truth table for function F = A XOR B"

1. Generate DFA with DiagramGenerator (format="dot")
2. Generate truth table in the web-interface sandbox
3. Embed or attach both outputs in response
```

### Dynamic Diagram Generation
```
Request: "Create a DFA for regular expression (0|1)*01"

1. Parse regex
2. Build automaton algorithm
3. Generate DOT spec programmatically
4. Render with DiagramGenerator
```

### Diagram Validation Pipeline
```
1. DiagramValidator(format, spec) → syntax check
2. If valid → DiagramGenerator(...) → render
3. If invalid → return error with suggestions
```
