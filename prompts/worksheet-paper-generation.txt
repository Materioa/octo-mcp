# Worksheet / Sample Paper Generation

## Intent

Generate polished worksheets, sample papers, question papers, and question banks using a reusable academic template. The default layout follows a formal exam-paper style with a centered institution header, metadata blocks, numbered instructions, section headings, and bordered question tables with marks, CO, and BT columns.

## Tools

- `CreateWorksheetPaper` — returns the final generation prompt once required paper settings are available.
- `CreateOrEditPaperTemplate` — creates or edits a reusable `Skill.md` template based on the user's described layout.

## Required Inputs

Before generation, collect:

1. Answer key: none, at last, or after each question.
2. Bloom's taxonomy: yes/no. If yes, levels to include: remember, understand, apply, analyse, evaluate, create.
3. Difficulty: easy, medium, or hard.
4. Question types and marks distribution: MCQ, fill blanks, short answer, long answer, case study, numericals, etc.
5. Output format: docx, pdf, or md.

## Grounding And Validation

Before generating questions, ground the content in available source material:

- If the request maps to Materio content, gather relevant syllabus and chapter/question-bank material using Materio tools such as `SnapSearch`, `DeepThink`, or `ResourceAccess`.
- Do not use `GlobalSearch` for content grounding; it is not a content retrieval tool for worksheets.
- If the user provides attached PDFs, notes, or other resources outside Materio, ground the output in those materials instead.
- Mixed-source mode is allowed, but the generated questions should stay faithful to the validated scope.
- Do not invent syllabus coverage or unsupported facts when source material is thin.

## Claude Modal Flow

When running in Claude and any required input is missing, call Claude's internal `ask_user_input_v0` tool as a modal or sequence of questions. After collecting answers, call `CreateWorksheetPaper` with the completed values.

Ask all required inputs before generating. Do not skip or infer missing fields.

Recommended modal questions:

1. Do you want an answer key? If yes, should it be at the end or after each question?
2. Should the paper follow Bloom's taxonomy? If yes, which levels should be included?
3. What difficulty should the paper use: easy, medium, or hard?
4. Which question types should be included, and what marks distribution should each type have?
5. Which output format should be created: docx, pdf, or md?

## Template Customization

If the user wants a custom format, call `CreateOrEditPaperTemplate`.

The user can describe:

- Header format, logo placement, title hierarchy, metadata fields, and alignment.
- Font family, font sizes, margins, spacing, and section styles.
- Body format, question table columns, marks placement, answer-key placement, and page-break rules.
- Separate variants for worksheet, sample paper, question paper, and question bank if needed.

The tool returns a `Skill.md` template. Keep that template in the same chat context and pass it to `CreateWorksheetPaper` as `templateSkillMd` for later worksheets, papers, or question banks. The user can also reuse the returned Markdown in a future chat to restore the template context.

## Quality Rules

- Keep the layout print-friendly.
- Make marks add up when total marks are provided.
- Keep questions age/level appropriate.
- Ground the output in validated source material before writing questions.
- If Bloom's taxonomy is enabled, include BT labels.
- Use CO labels consistently when the template includes a CO column.
- Match answer-key placement exactly to the user's selected preference.
- For diagram-type questions, do not describe the diagram in text. Generate the diagram using the platform sandbox (Python tools such as graphviz, networkx, schemdraw, matplotlib) and embed the resulting image in the document.
